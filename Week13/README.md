学习笔记


#### 毕业总结
13周过去悄悄过去,最开始报课我其实是犹豫的。因为本身我是
不太懂Go这门语言的，所以对于报课，特别是对于我第一次报这种训练营来说还是有些忐忑，不知道是否打水漂。  
一开始也只是以为毛大的课程是对于go 语言的讲解。但其实一直跟下来来说，并不是这样的。更多的带给我们的是工程的实践，组建的选择，分析每个组件的原理，如果更好的使用。一种方法轮的实践。      
对于毛大的课程来说，这么点时间只能说学习大概，在日常工作工程实践中需要不断的去实践，剩下的时间可以去二刷三刷再去回味。   
整个课程下来，不仅分享了原理讲解还包括很多的案例分析、工程实践、代码实践。带给我们更应该是带着思考去看我们的工程项目去比较。    
第一周的微服务实践特别有感触，像是回顾我在之前在公司的过程,从单体是架构，到微服务拆分，以前只是按照自己的想法感觉。又有系统的学习过微服务架构，只是觉得这样子更好。知其然知其所以然，但是毛大的课程对我来说豁然开朗。  
之前只知道限流怎么用熔断怎么用，稍微知道其原理，毛大带我领略了其中的算法思路，一通百通，还活学活用的教我们为什么使用，  
之后的异常处理对我来说感受很多，因为之前没写过Go在这里第一次实践go语言还是很有意思的。  
之后的工程化实践对我来说也是很宝贵，让我知道一线的架构师在设计时的思路。
#### 结束语
之后的日子也是需要一直学习的，程序员的道路就是不断学习不断实践，不能总是抱着老旧的知识点，更不能因为不去深究知识点。  
对于不懂的知识点一定要学习深挖。这是我在毛大身上看到的，对于一些组件使用信手捏来，对于代码处理和工程实践总是能说的头头是道。 
最后自学go的人来说，语言语法可以看两次就知道，主要是在于实践。希望在这年中我能找到机会去实践go语言，希望训练营的伙伴工作顺利。
